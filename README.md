## 这是什么

VNDB Profile Search 是一个用来检索 VNDB 作品、角色、tag 和 trait 的网页工具。

它更像是一个“本地索引 + 相似推荐”的搜索页面：你可以先搜索作品或角色，把感兴趣的项目加入样本，然后让页面根据这些样本的标签画像找出相似作品或相似角色。你也可以直接用 VN tag 和角色 trait 组合检索，例如寻找某类题材作品里的某类角色。

## 主要能做什么

- 搜索 VN 作品。
- 搜索角色。
- 根据已选择的作品推荐相似作品。
- 根据已选择的角色推荐相似角色。
- 直接选择 VN tag 搜索作品。
- 直接选择角色 trait 搜索角色。
- 同时选择 VN tag 和角色 trait，搜索“作品 + 角色”的组合结果。
- 按评分、投票数、相关度、置信度等方式排序。
- 默认隐藏 R18 与剧透内容，需要时可手动开启显示。

## 数据来自哪里

数据主要来自 VNDB。

本项目会把 VNDB 提供的数据整理成一个本地索引文件，网页加载这个索引后就能在浏览器里进行搜索。作品和角色的图片、简介等详细信息不会一开始就全部请求，只有在你点击“载入当前页详情”或把项目加入样本后，页面才会向 VNDB 请求对应详情。

VNDB 数据遵循 VNDB 的数据库许可。图片、简介、评分、投票数和原始条目信息都以 VNDB 原站为准。

## 如何使用

### 作品模式

在“VN 模式”里输入作品名、别名或 VNDB 编号，例如 `v12984`。搜索后可以把作品加入样本。加入复数个样本后，页面会根据这些作品的 tag 生成一个画像，并推荐相似作品。

### 角色模式

在“角色模式”里输入角色名或角色编号，例如 `c35176`。搜索后可以把角色加入样本。加入复数个样本后，页面会根据角色 trait 生成画像，并推荐相似角色。

如果直接搜角色名没有结果，会尝试把输入内容当作作品名再搜索一次，然后列出匹配作品里的角色。

### tag 检索

在“tag 检索”里可以直接从树状列表中选择 VN tag 和角色 trait。

只选择 VN tag 时，会搜索作品。

只选择角色 trait 时，会搜索角色。

同时选择 VN tag 和角色 trait 时，会搜索符合条件的“作品 + 角色”组合。例如可以尝试寻找“超自然题材作品中的蓝发角色”。

## 搜索算法说明

### 1. 本地文本搜索

网页加载时会读取一个已经生成好的本地索引。这个索引里包含作品标题、别名、厂商、角色名、关联作品、tag、trait 等基础信息。

普通搜索会先在本地索引里匹配文本。VN 搜索会看作品编号、标题、别名和厂商等字段；角色搜索会看角色编号、角色名和关联作品。输入 VNDB 编号时会优先直接匹配编号。

这样做的好处是搜索速度快，也不会因为每次输入都请求 VNDB API 而触发请求限制。

### 2. 样本画像

当你把作品或角色加入样本后，页面会把它们的 tag 或 trait 合并成一个“画像”。

对 VN 来说，画像主要来自作品 tag。VNDB 的 tag 本身带有置信度或权重，权重越高，表示这个 tag 越能代表该作品。

对角色来说，画像主要来自角色 trait。多个角色样本会把共同或相近的 trait 聚合起来，形成一个角色特征画像。

之后页面会拿这个画像去和数据库中的其他作品或角色比较，计算相似度并排序。

### 3. 权重与搜索前 N 个标签

画像中可能有很多 tag 或 trait，但不是每一个都同样重要。页面会优先使用权重最高的一部分参与搜索，也就是界面里的“搜索前 N 个 tag / trait”。

这个数量越大，搜索条件越丰富，但也可能引入噪声；数量越小，搜索更集中，但可能漏掉一些相似项目。

### 4. 重点 tag / trait

你可以点击 tag 或 trait，把它标记为重点。重点项会获得更高权重，并且不占用“搜索前 N 个”的数量限制。

搜索时会优先尝试满足所有重点项。如果完全满足的结果太少，页面会逐步放宽一部分重点条件，以便补充候选结果。结果中会显示重点覆盖情况，也就是这个结果命中了多少你标记的重点项。

这可以避免一个常见问题：条件太严格时完全搜不到东西，条件太松时又出现大量无关结果。

### 5. 父标签与子标签

VNDB 的 tag 和 trait 有层级结构。例如一个大类下面可能包含很多更具体的子标签。

页面展示时会保留这种层级。选择父标签时，搜索会把它下面的有效子标签也作为候选条件考虑；选择子标签时，算法也会适当考虑它的父级关系，让相近标签之间更容易互相匹配。

这样做可以让搜索既支持精确条件，也支持比较宽泛的题材或特征条件。

### 6. R18、剧透、technical 与 Character/Scene 标签

默认情况下，R18、剧透内容，以及 VNDB 中 Character / Scene 两个子树下的作品 tag 不会参与搜索。

规则大致如下：

- R18 tag/trait 默认不参与搜索。
- 只有你明确把某个 R18 tag/trait 标记为重点时，它才会参与搜索。
- 剧透 tag/trait 默认不参与搜索。
- 开启“允许剧透标签”后，剧透 tag/trait 才会参与搜索，无论你是否选中它们。
- technical tag 默认不显示，也不参与 VN 搜索。
- 开启“显示技术性标签”后，technical tag 会以橙色底色显示在 VN 标签列表中；即使开启显示，它们仍然只用于查看，只有被你明确标记为重点时才参与搜索。
- Character / Scene 相关 VN tag 默认不显示，也不参与 VN 聚合搜索。
- 开启“显示 Character/Scene 标签”后，这些 tag 会出现在界面中，但仍然只用于查看；只有被你明确标记为重点时，它们才会参与搜索。

这样做是为了避免搜索结果因为隐藏标签、剧透标签、技术性标签或偏角色/场景说明的作品 tag 而变得不可控。

页面中的标签颜色大致含义如下：

- 天蓝色：默认会参与画像追踪/检索的普通 tag/trait。
- 粉色：R18 / sexual tag/trait；默认不参与搜索，除非被明确标记为重点。
- 黄色：低等级剧透 tag/trait。
- 红色：高等级剧透 tag/trait。
- 橙色：technical tag；默认隐藏，开启“显示技术性标签”后显示；不参与 VN 搜索，除非被标记为重点。
- 靛紫色：Character / Scene 相关 VN tag；默认隐藏，开启显示后也只用于查看，除非被标记为重点。
- 青绿色或深蓝色：已经标记为重点的 tag/trait。
- 紫色：它下面有子标签已被标记为重点的父级标签。
- 高亮黄蓝色：在 tag/trait 搜索框中命中的候选标签。

如果一个标签同时属于多种类型，页面会优先显示更需要提醒的状态，或使用混合色表示叠加状态。

### 7. 直接 tag 检索

tag 检索模式和样本推荐模式不完全一样。

样本推荐是先根据已选样本生成画像，再根据画像找相似项目。

tag 检索则是你直接指定想要的 tag/trait，页面按照这些条件去筛选数据库。如果同时选择了作品 tag 和角色 trait，页面会先分别寻找符合条件的作品与角色，再组合出符合条件的结果。

如果重点条件过多导致结果太少，页面会尝试降低一部分条件的严格程度，尽量给出可浏览的候选。

### 8. 排序

搜索结果可以按不同方式排序：

- 相关值：结果和当前画像或条件的相似程度。
- 置信度：重点条件的命中情况。
- VN rating：VNDB 中的作品评分。
- 投票数：VNDB 中的投票数量。
- 标题：按标题排序。

角色搜索还可以选择是否按关联 VN 平均分加权。开启后，来自高评分作品的角色会更容易排在前面。注意：由于VNDB部分作品的投票人数并不完全均等，可能产生意想不到的结果。

### 9. 分页和详情加载

页面不会一次性请求所有结果的图片和简介。搜索结果会分页显示，只有点击“载入当前页详情”后才会请求当前页条目的详情。

这样能减少网络请求，也能避免 VNDB API 请求过快。

## 本地启动

先安装依赖：

```bash
npm ci
```

启动开发服务器：

```bash
npm run dev
```

命令运行后，终端会显示一个本地地址，通常类似：

```text
http://localhost:5173/
```

## 构建

生成正式静态文件：

```bash
npm run build
```

构建结果会输出到：

```text
dist
```

预览构建结果：

```bash
npm run preview
```

## 注意事项

- 第一次加载可能需要一点时间，因为本地索引文件较大。
- 搜索结果是基于本项目的本地算法，不等同于 VNDB 官方搜索排序。
- VNDB 的评分、投票数、图片和简介可能会更新，页面内容取决于最近一次数据生成时间。
- tag/trait 的中文和日文翻译由项目维护，新增条目在补全翻译前可能会显示英文。
- 默认隐藏 R18 与剧透，但这不代表页面能替代内容分级判断。
